# PART 1: Vibration Classification

## Overview

This part of the project implements a **real-time vibration classification system** on the **Arduino Nano 33 BLE** using **TensorFlow Lite for Microcontrollers (TinyML)**.

The system uses the onboard **IMU sensor (LSM9DS1)** to detect and classify vibrations into three categories:

- **idle**: no vibration detected  
- **weak**: vibration generated by **one smartphone**  
- **strong**: vibration generated by **two smartphones simultaneously**

The goal of this project is to demonstrate a complete **TinyML pipeline**, from sensor data collection to real-time inference directly on a microcontroller.

---

## Workflow

The project follows a three-step pipeline:

### 1. Data Collection (Step 1-DataGeneration_Arduino)

- The **Arduino Nano 33 BLE** collects raw data from the IMU sensor (accelerometer + gyroscope)
- The captured measurements are:
  - `aX`, `aY`, `aZ` (acceleration)
  - `gX`, `gY`, `gZ` (gyroscope)
- The system captures **100 samples per vibration**
- Sampling frequency: **~50 Hz**
- Each vibration window lasts approximately **2 seconds**
- Data is transmitted via **serial communication** to a Python script
- The measurements are saved as **CSV files**

In this project:

- **weak vibration**: vibration produced by **one smartphone**
- **strong vibration**: vibration produced by **two smartphones simultaneously**

---

### 2. Model Training (Step 2-Training)

A **Python Jupyter Notebook** is used to train the classification model.

The training pipeline includes:

- Loading CSV dataset files
- Visualizing IMU signals
- Extracting statistical features
- Preparing the dataset
- Training a neural network model

#### Model Conversion

After training, the model is:

1. Converted to **TensorFlow Lite format**

2. Converted into a **C++ header file**

---

### 3. Real-Time Inference (Step 3-Inference_Arduino)

The Arduino sketch loads the TinyML model and performs vibration classification in real time.

The process is as follows:

1. Capture **100 IMU samples**
2. Compute vibration features
3. Normalize the features
4. Run inference using **TensorFlow Lite Micro**
5. Display the predicted class through the **Serial Monitor**

Each prediction corresponds to **one vibration window (~2 seconds)**.

## Quick Start

1. **Collect training data**

   - Upload the `data_generation.ino` sketch
   - Generate IMU vibration data
   - Save the recordings as CSV files

2. **Train the model**

   - Run the `training_vibrations.ipynb` notebook
   - Train the neural network
   - Generate the TensorFlow Lite model

3. **Deploy on Arduino**

   - Include `model_data.h`
   - Upload `inference_vibrations.ino`
   - Observe predictions in the **Serial Monitor**

The system will then classify vibrations **in real time directly on the Arduino board**.
