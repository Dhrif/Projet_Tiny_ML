# TinyML Vibration Classification System
## Arduino Nano 33 BLE – IMU Based Vibration Detection

---

# 1. Project Overview

This project implements a **TinyML vibration classification system** running directly on an **Arduino Nano 33 BLE**.

The system uses the onboard **IMU sensor (LSM9DS1)** to detect vibrations and classify them into three categories using a **neural network deployed with TensorFlow Lite for Microcontrollers**.

The classification is performed **entirely on the microcontroller**, without requiring a computer.

---

# 2. Vibration Classes

Three vibration categories are used in this project.

| Class | Description |
|-----|-----|
| idle | No vibration |
| weak | Vibration generated by **one smartphone** |
| strong | Vibration generated by **two smartphones simultaneously** |

### Experimental Setup

- **Weak vibration** → one smartphone placed near the Arduino with vibration enabled.
- **Strong vibration** → two smartphones vibrating simultaneously near the Arduino.

This approach allows generating vibrations with **different intensities**.

# 6. Step 1 — Data Generation

## Objective

Collect vibration data from the IMU sensor.

---

## Arduino data generation sketch

Upload the following sketch to the Arduino:

1-DataGeneration_Arduino/data_generation.ino

This program:

- reads IMU data
- sends measurements via Serial
- outputs the following values:
aX,aY,aZ,gX,gY,gZ


---

## Sampling configuration

- Sampling rate: **50 Hz**
- Window size: **100 samples**
- Duration per vibration: **2 seconds**

---

## Data collection procedure

### Idle class

1. Run the Arduino program.
2. Leave the board untouched.
3. Record sensor data.

Save the recorded data as:
vibration_idle.csv


---

### Weak vibration

1. Place **one smartphone** near the Arduino board.
2. Enable vibration (incoming call or vibration mode).
3. Record IMU data.

Save the recorded data as:
weak.csv


---

### Strong vibration

1. Place **two smartphones** near the Arduino board.
2. Enable vibration on both phones simultaneously.
3. Record IMU data.

Save the recorded data as:
strong.csv


---

# 7. Step 2 — Dataset Creation

The CSV files contain raw IMU data with the following format:
aX,aY,aZ,gX,gY,gZ


Example rows:
0.01,0.98,-0.02,0.12,0.03,-0.02
0.02,1.01,-0.01,0.10,0.04,-0.01


Each row corresponds to **one IMU measurement**.

---

# 9. Feature Extraction

Instead of feeding raw signals to the neural network, statistical features are computed.

For each **100-sample window**, the magnitude of the acceleration and gyroscope signals is calculated.

### Acceleration magnitude
a_mag = sqrt(aX² + aY² + aZ²)

### Gyroscope magnitude
g_mag = sqrt(gX² + gY² + gZ²)


---

## Extracted Features

For both acceleration and gyroscope signals, the following features are computed:

- RMS (Root Mean Square)
- Variance
- Energy

The final feature vector is:
[rms_acc,
var_acc,
energy_acc,
rms_gyro,
var_gyro,
energy_gyro]


Each window of **100 IMU samples** becomes **one training sample with 6 features**.

---

# 9. Dataset Preparation

After feature extraction:

1. The dataset is **randomly shuffled**.
2. The dataset is split into three subsets.

| Dataset | Percentage |
|-------|-----------|
| Training | 60% |
| Validation | 20% |
| Test | 20% |

---

# 10. Data Normalization

The input features are normalized using:
x_normalized = (x - mean) / std

Where:

- **mean** = mean value of each feature from the training set
- **std** = standard deviation of each feature

The parameters are saved as:
models/  
├── mean.npy  
└── std.npy  

These parameters are reused during **Arduino inference**.

---

# 11. Neural Network Model

A lightweight neural network is used for classification.

Architecture:
nput layer: 6 features

Dense layer: 20 neurons (ReLU)
Dense layer: 10 neurons (ReLU)

Output layer: 3 neurons (Softmax)


The network predicts the probability of each vibration class.

---

# 12. Model Training

Training configuration:
optimizer: Adam
loss function: categorical_crossentropy
epochs: 90
batch size: 8


During training, the notebook plots:

- training accuracy
- validation accuracy
- training loss
- validation loss

These plots help monitor model performance.

---

# 13. TensorFlow Lite Conversion

After training, the model is converted to **TensorFlow Lite format**.
vibration_model.tflite

This format is optimized for embedded devices.

The model is stored in:
2-Training/models/


---

# 14. Arduino Model Integration

Since Arduino cannot directly load `.tflite` files from disk, the model is converted into a **C++ header file**.
model_data.h


This file contains the neural network weights encoded as a byte array.

---

# 15. Step 3 — Embedded Inference

Upload the following sketch:
3-Inference_Arduino/inference_vibrations.ino


The Arduino performs the following steps:

1. Collect 100 IMU samples
2. Compute vibration features
3. Normalize the features
4. Run TensorFlow Lite Micro inference
5. Output the predicted class via Serial

Example output:
Prediction: idle
Prediction: weak
Prediction: strong


Each prediction corresponds to **one vibration window (~2 seconds)**.
